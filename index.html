Capítulo 5: Formato de texto

Si bien la mayoría de las etiquetas HTML se utilizan para crear elementos, HTML también proporciona etiquetas de formato de texto para aplicar estilos relacionados con el texto a porciones de texto. Este tema incluye ejemplos de formato de texto HTML, como resaltado, texto en negrita, subrayado, subíndice y tachado.

Sección 5.1: Resaltado

El elemento <mark> es nuevo en HTML5 y se utiliza para marcar o resaltar texto en un documento "debido a su relevancia en otro contexto ".[1].

El ejemplo más común sería en los resultados de una búsqueda donde el usuario ha ingresado una consulta de búsqueda y los resultados se muestran resaltando la consulta deseada.

<(p)>Aquí hay contenido de un artículo que incluye el <(mark)>consulta buscada </mark>que estamos buscando. Resaltar el texto hará que sea más fácil para el usuario encontrar lo que están buscando.</p>

Salida:

Aquí hay contenido de un artículo que contiene la consulta buscada que estamos buscando. Resaltar el texto facilitará al usuario encontrar lo que busca.

Un formato estándar común es el texto negro sobre un fondo amarillo, pero esto se puede cambiar con CSS.

Sección 5.2: Negrita, cursiva y subrayado

Texto en negrita

Para texto en negrita, use el <strong> o <b> etiquetas:

<strong> Texto en negrita aquí </strong>

o

<b> Texto en negrita aquí en negrita </b>

¿Cual es la diferencia? Semántica. <strong> se utiliza para indicar que el texto es fundamental o semánticamente importante para el texto circundante, mientras que<b> no indica tal importancia y simplemente representa texto que debería estar en negrita.

Si tuvieras que usar <b> Un programa de conversión de texto a voz no diría la (s) palabra (s) de manera diferente a cualquiera de las otras palabras a su alrededor: simplemente está llamando la atención sobre ellas sin agregar ninguna importancia adicional. Mediante el uso <strong>, Sin embargo, el mismo programa querría decir esas palabras con un tono de voz diferente para transmitir que el texto es importante de alguna manera.

Texto en cursiva

Para poner en cursiva el texto, use el <em> o <i> etiquetas:

<em>Texto en cursiva aquí</em>

O

<i>Texto en cursiva aquí</i>

¿Cual es la diferencia? Semántica.<em> se utiliza para indicar que el texto debe tener un énfasis adicional que debe ser estresado, mientras <i> simplemente representa texto que debe diferenciarse del texto normal que lo rodea.

Por ejemplo, si desea enfatizar la acción dentro de una oración, puede hacerlo enfatizándola en cursiva a través de <em>:"¿Podrías enviar la edición ya?"

Pero si estuviera identificando un libro o periódico que normalmente pondría en cursiva estilísticamente, simplemente usaría <i>: "Me vi obligado a leer Romeo y Julieta en la escuela secundaria.

Texto subrayado

Mientras que la<u>elemento en sí fue obsoleto en HTMl 4, se reintrodujo con un significado semántico alternativo en HTML 5: para representar una anotación no textual no articulada. Puede usar una representación de este tipo para indicar texto mal escrito en la página, o para una marca de nombre propio en chino.

<p>Este párrafo contiene algunos<u> texto.</p>

Sección 5.3: Abreviatura

Para marcar alguna expresión como abreviatura, utilice<abbr>etiqueta:

<p> me gusta escribir <abbr title="Hypertext Markup Language">!

Si está presente, el atributo de título se utiliza para presentar la descripción completa de dicha abreviatura.

Sección 5.4: Insertado, eliminado o golpeado

Para marcar texto como insertado, use el <ins>etiqueta:

<ins> Nuevo texto</ins>

Para marcar el texto como eliminado, use el<del>

<del>Texto eliminado</del>

Para tachar el texto, use el<s>etiqueta:

<s>Texto de tachado aquí</s>

Sección 5.5: Superíndice y subíndice

Para desplazar el texto hacia arriba o hacia abajo, puede utilizar las etiquetas<sup> y <sub>

Para crear superíndice:

<sup>superíndice aquí</sup>

Para crear un subíndice:

<sub>subíndice aquí</sub>

Capítulo 11: Atributos de datos

Sección 11.1: Compatibilidad con navegadores antiguos

Los atributos de datos se introdujeron en HTML5, que es compatible con todos los navegadores modernos, pero los navegadores más antiguos antes HTML5 no reconoce los atributos de los datos.

Sin embargo, en las especificaciones HTML, los atributos que no son reconocidos por el navegador deben dejarse en paz y el navegador simplemente los ignorará al representar la página.

Los desarrolladores web han utilizado este hecho para crear atributos no estándar que son atributos que no forman parte del Especificaciones HTML. Por ejemplo, el atributo de valor en la línea siguiente se considera un atributo no estánda porque las especificaciones para el<img>la etiqueta no tiene un atributo de valor y no es un atributo global:

<img src="sample.jpg" value="test"/>

Esto significa que aunque los atributos de datos no son compatibles con los navegadores más antiguos, aún funcionan y puede configurar y recuperarlos usando los mismos métodos genéricos setAttribute y getAttribute de JavaScript, pero no puede usar el nueva propiedad del conjunto de datos que solo es compatible con los navegadores modernos.

Sección 11.2: Uso de atributos de datos

Los atributos de datos HTML5 * proporcionan una forma conveniente de almacenar datos en elementos HTML. Los datos almacenados se pueden leer o modificado usando JavaScript

<div data-submitted="yes" class="user_profile">

... algo de contenido ...

</div>

La estructura del atributo de datos es data- *, es decir, el nombre del atributo de datos viene después de la parte de datos. Usando esto nombre, se puede acceder al atributo.

Los datos en formato de cadena (incluido json) se pueden almacenar utilizando el atributo data- *.

Capítulo 21: Barras de navegación

Sección 21.1: Barra de navegación básica

Las barras de navegación son esencialmente una lista de enlaces, por lo que los elementos ul y li se utilizan para encerrar enlaces de navegación.

<url>

<li><a href="#">Hogar</a></li>

<li><a href="#">Acerca de</a></li>

<li><a href="#">Contacto</a></li>

</url>

Sección 21.2: Barra de navegación HTML5

Para hacer una barra de navegación usando el elemento de navegación HTML5, encierre los enlaces dentro de la etiqueta de navegación.

<nav>

<a href="#">Hogar</a>

<a href="#">Acerca de</a>

<a href="#">Contacto</a>

</nav>

Capítulo 2: Estructura y formato de una regla CSS

Sección 2.1: Listas de propiedades

Algunas propiedades pueden tomar varios valores, conocidos colectivamente como lista de propiedades.

/* Dos valores en esta lista de propiedades */

span {

text-shadow: yellow 0 0 3px, green 4px 4px 10px;

}

/* Formato alternativo */

span {

text-shadow:

yellow 0 0 3px,

green 4px 4px 10px;

}

Sección 2.2: Selectores múltiples

Cuando agrupa selectores CSS, aplica los mismos estilos a varios elementos diferentes sin repetir los estilos en tu hoja de estilo. Utilice una coma para separar varios selectores agrupados.

div, p { color: blue }

Entonces el color azul se aplica a todos <div> elementos y todo <p> elementos. Sin la coma solamente <p> elementos que son Un hijo de un <div> Sería rojo.

Esto también se aplica a todos los tipos de selectores.

p, .blue, #first, div span{ color : blue }

Esta regla se aplica a:

<p>

elementos de la clase azul

elemento con el ID primero

cada <span> dentro de un <div>

Sección 2.3: Reglas, selectores y bloques de declaración

Una regla CSS consta de un selector (por ejemplo, h1) y un bloque de declaración ({}).

h1 {}

Capítulo 19: Opacidad

La opacidad de un elemento se puede establecer mediante la propiedad de opacidad. Los valores pueden oscilar entre 0,0 (transparente) y 1,0 (opaco).

Ejemplo de uso

<div style="opacity:0.8;">

Valor de propiedad Transparencia
opacidad: 1.0; Opaco
opacidad: 0.75; 25% transparente (75% Opaco)
opacidad: 0.5; 50% transparente (50% Opaco)
opacidad: 0.25; 75% transparente (25% Opaco)
opacidad: 0.0; Transparente

Sección 19.2: Compatibilidad de IE para `opacidad`

Para usar la opacidad en todas las versiones de IE, el orden es:

.transparent-element {

/* for IE 8 & 9 */

-ms-filter:"progid:DXImageTransform.Microsoft.Alpha(Opacity=60)"; // IE8

/* works in IE 8 & 9 too, but also 5, 6, 7 */

filter: alpha(opacity=60); // IE 5-7

/* Modern Browsers */

opacity: 0.6;

}
